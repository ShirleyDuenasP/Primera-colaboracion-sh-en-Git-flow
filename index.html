Introducción Asignando roles para trabajo en grupo

Descripción del Proyecto

Este proyecto tiene como objetivo dar a conocer algunos de los temas relevantes vistos en clase, mientras se hace uso de la herramienta git flow para colaborar en proyectos grupales, con el fin de adquirir conocimientos sobre cómo asignar roles y tareas, entre otras funciones.

Roles del Proyecto

Estos son los roles que se manejarán en el trabajo:

  • Project Manager (PM): Encargado de gestionar el repositorio, organizar las tareas y revisar las pull requests.
  • Desarrollador: Responsable de desarrollar funcionalidades, hacer forks del repositorio y enviar pull requests para revisión.

Flujo de Trabajo con Git

  • Hacer un fork del repositorio principal.
  • Clonar el repositorio de forma local.
  • Crear ramas descriptivas para cada tarea asignada.
  • Realizar commits frecuentemente siguiendo buenas prácticas.
  • Enviar pull requests y esperar revisión antes de fusionar los cambios.

Asignando roles para trabajo en grupo

Para este trabajo es muy importante mantener un orden y condinacion, implementando buenas practicas por parte de los desarrolladores esto con el fin de lograr una culminacion del trabajo de forma organizada.

Por otro lado el PM es la persona que se encarga de mantener el orden de las tareas y su asignacion, este tambien debe supervisar cada uno de los cambios implementados.

  • Project Manager (PM):
    • Definir los objetivos del proyecto.
    • Crear o reutilizar un repositorio en GitHub y estructurar el proyecto.
    • Organizar tareas, gestionar el cronograma y hacer un seguimiento del progreso.
    • Revisar y fusionar las pull requests de los desarrolladores.
  • Desarrollador:
    • Hacer un fork del repositorio principal y trabajar en su propio repositorio.
    • Desarrollar nuevas funcionalidades o corregir errores asignados.
    • Enviar pull requests al Project Manager para revisión.

Es importante que todos los estudiantes participen activamente en sus roles asignados para garantizar el éxito del proyecto.
